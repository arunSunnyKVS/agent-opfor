How it works

Three steps to find vulnerabilities

From target description to a full vulnerability report in minutes. Same workflow whether you use the CLI, an AI agent, or MCP inside Cursor.

1

Configure

Describe your target — an HTTP endpoint, a local script, or a live MCP server. Pick your LLM provider and attack suite.

opfor setup
2

Generate

Opfor synthesises OWASP-mapped attack prompts tailored to your target description and saves them for inspection.

opfor generate
3

Run & Report

Attacks fire at your target. An LLM judge evaluates every response and produces an HTML + JSON report.

opfor run
Three modes

Fits your existing workflow

All three modes share the same evaluators, attack templates, and judge logic. Pick the one that fits where you work.

CLI

Command-line tool

Run opfor directly in your terminal or pipe it into CI/CD. Deterministic runs with reproducible attack plans and logged results.

opfor setup opfor generate opfor run opfor init
Best for terminal-first workflows and CI/CD pipelines.
MCP Server

Agents that test agents

Add opfor as an MCP server in Cursor or Claude Desktop. Ask in chat — your AI assistant calls the security tools for you.

opfor_list_evaluators opfor_setup opfor_run
Best for Cursor / Claude Desktop workflows.
Skills

Slash commands

Use /opfor-setup and /opfor-run in any AI coding agent — Cursor, Claude Code, Windsurf, and more.

/opfor-setup /opfor-run
Best for any AI coding agent with skill/slash-command support.

Quick start

Up and running in minutes

Clone, install, build, and start finding vulnerabilities. Supports OpenAI, Anthropic, Groq (free tier), and Google — bring whichever key you have.

OpenAI Anthropic Groq Google + any OpenAI-compatible
Full install guide → CLI docs
bash
# 1. Clone and install
git clone https://github.com/KeyValueSoftwareSystems/opfor.git
cd opfor
npm install --ignore-scripts
npm run build
npm install -g ./cli
 
# 2. Set your LLM API key
export GROQ_API_KEY=your-key-here
# or OPENAI_API_KEY, ANTHROPIC_API_KEY…
 
# 3. Run
opfor setup # interactive wizard
opfor run --attacks .opfor/attacks/latest.json
opfor.config.json
{
"llm": { "provider": "groq", "model": "llama-3.3-70b-versatile" },
"target": {
"name": "My Support Bot",
"type": "http-endpoint",
"endpoint": "http://localhost:4000/chat"
},
"selection": { "mode": "suite", "suite": "owasp-llm-top10" }
}
Features

Everything you need for AI security testing

🎯

LLM-as-judge

Every response is evaluated by a language model with configurable pass/fail criteria — far more accurate than regex-based checks.

🔄

Multi-turn attacks

Adversarial conversations where each follow-up escalates the attack — mirroring how real attackers probe defences.

📊

HTML + JSON reports

Human-readable HTML report for review, machine-readable JSON for CI/CD. Every request, response, and verdict is logged.

⚡

Telemetry-grounded

Enrich attacks with real production traces via Langfuse or Netra integration — target the actual paths your users take.

🧩

Extensible evaluators

Add new attack patterns as plain Markdown files with YAML frontmatter. No TypeScript, no build step, no pull request required.

🔁

CI/CD ready

Deterministic, reproducible runs with fixed attack plans. Integrate into GitHub Actions, GitLab CI, or any pipeline.
